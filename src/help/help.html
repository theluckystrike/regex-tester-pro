Regex Flags
Flags control how the regex engine processes your pattern. Toggle them with a single click on the flag buttons next to the pattern input.
g Global
Find ALL matches in the text, not just the first one
i Case Insensitive
Match uppercase and lowercase letters interchangeably
m Multiline
Make ^ and $ match the start and end of each line
s Dotall
Make the dot (.) also match newline characters
u Unicode
Enable full Unicode support for international text
y Sticky
Match only from the current position in the text

Capture Groups
When your regex contains parentheses for capture groups, the extension automatically extracts and displays each group separately. Both numbered groups and named groups (using (?<name>...)) are fully supported.
Pattern (?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2}) Test Today is 2026-02-08 Groups year = 2026 month = 02 day = 08

Pro Tips . Get Maximum Value

Five expert workflows that will transform how you work with regular expressions.

üèÜ
Workflow 1 . Validate Before You Ship
Before deploying a regex to production, paste real-world edge cases into the test string. Include empty strings, unicode characters, special symbols, and boundary values. The live highlighting will reveal false positives and missed matches instantly.
‚ú®
Workflow 2 . AI Generate then Refine then Save
Use AI Generate to get a starting pattern. Test it against your data. Tweak specific parts while watching the explainer to understand what each token does. Once perfected, save the pattern to your history for instant reuse.
üìö
Workflow 3 . Learn by Modifying Examples
Open the Examples Library and load a sample. Study the pattern structure through the explainer. Then start modifying parts of the pattern and observe how the matches change. This builds regex muscle memory significantly faster than reading tutorials.
üîÅ
Workflow 4 . Batch Test with Replace
Open Find + Replace to preview how your regex transforms text. This is perfect for log parsing, data cleaning, and code refactoring tasks. See the transformed output live before writing a single line of code in your project.
‚ö°
Workflow 5 . Right-Click then Test then Export
Select text from any webpage or documentation. Right-click and test with Regex Tester Pro. Build your pattern with the live preview. Then use Code Export to get copy-ready code in your target language. The entire workflow takes under 30 seconds.
üî• Avoid common mistakes
Always test with the g flag to see ALL matches, not just the first
Use the m flag when testing multiline content like logs or CSV data
Remember to escape special characters with a backslash
Use the cheat sheet (üìã icon in header) when you forget syntax
